# ClimatESP
Система Gemini CLI устроена так, что память строго изолирована для каждого проекта:

   1. Локальная память проекта (Private Project Memory): Тот файл MEMORY.md, который я создал, находится в папке,
      привязанной именно к этому проекту (.../tmp/climat-esp8266/). Когда вы откроете другой проект, я загружу память
      именно из его папки. Информация об инкубаторе никогда не "всплывет" в проекте, например, умного дома или
      мобильного приложения.
   2. Глобальная память (Global Personal Memory): Существует еще файл C:/Users/Hospes/.gemini/GEMINI.md. Туда я
      записываю только ваши общие предпочтения (например, "всегда отвечай на русском" или "я предпочитаю библиотеку
      ArduinoJson"). Это единственное, что общее для всех проектов.
   3. Инструкции проекта (GEMINI.md в корне): Если вы захотите, чтобы не только я, но и любой другой ИИ-помощник (или
      ваш коллега) знал правила этого проекта, такие инструкции пишутся в файл GEMINI.md прямо в папке проекта и
      фиксируются в Git.

  Итог: Ваши проекты для меня — это разные "комнаты". Заходя в комнату "Climat_ESP8266", я беру блокнот, который лежит
  именно в ней, и не смешиваю его с записями из других комнат.

  Если вы захотите перенести какую-то настройку во все будущие проекты — скажите мне, и я запишу это в "глобальный"
  блокнот.
