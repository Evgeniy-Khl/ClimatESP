## 3. Оптимизация прошивки и надежность (C++ / ESP8266) [ВЫПОЛНЕНО]
Аппаратная часть инкубатора работает в непрерывном режиме неделями. Любой сбой ПО или зависание шины I2C может погубить инкубационный материал.

*   **Интеграция I2C Bus Recovery:**
    *   Внедрить функцию `recoverI2C()` для автоматического "выбивания" зависших ведомых устройств с линий SDA/SCL путем генерации импульсов синхронизации вручную (успешно обкатано на проекте Quadus).
    *   Интегрировать проверку ошибок I2C в функции чтения/записи PCF8574 и производить перезапуск шины при сбоях.
*   **Аппаратный Watchdog (WDT):**
    *   Включить встроенный аппаратный сторожевой таймер ESP8266 на 5 секунд (`ESP.wdtEnable(5000);`) в `setup()` и регулярно сбрасывать его в `loop()` через `ESP.wdtFeed();`. Это спасет от полного зависания при сбое стека Wi-Fi или I2C.
*   **Переход с EEPROM на LittleFS (`config.json`):**
    *   Хранение настроек в сыром бинарном виде в EEPROM усложняет масштабирование проекта (при добавлении новой настройки съезжают адреса памяти).
    *   **Решение:** Хранить конфигурацию в виде JSON-файла в LittleFS. Это позволит легко добавлять любые новые параметры, а также делать бекапы настроек через веб-интерфейс.
*   **Оптимизация оперативной памяти (RAM):**
    *   Заменить динамическое выделение объектов `String` при формировании JSON-ответов на потоковую сериализацию непосредственно в сетевой клиент: `serializeJson(doc, client)`. Это предотвратит фрагментацию кучи (heap fragmentation) и случайные сбросы контроллера.

### 5. Чтение значений с выводов P6 и P7 [ВЫПОЛНЕНО]
  Для получения сигналов с внешних выводов вы можете считывать состояние микросхемы с помощью функции main.cpp:

  #### Вариант 1 (через побитовые операции):
    byte portState = readPCF8574();

    // Извлекаем состояние 6-го и 7-го выводов (нумерация битов с 0)
    bool p6_high = (portState & (1 << 6)) != 0; // true — на P6 уровень HIGH (открыт), false — притянут к LOW
    bool p7_high = (portState & (1 << 7)) != 0; // true — на P7 уровень HIGH (открыт), false — притянут к LOW

  #### Вариант 2 (через структуру  union Byte ):

  Вы можете использовать объявленную в main.h структуру  union Byte :

    union Byte pcfInput;
    pcfInput.value = readPCF8574();

    bool p6_high = pcfInput.bitfield.a6; // Состояние P6
    bool p7_high = pcfInput.bitfield.a7; // Состояние P7

  • Примечание: Уровень  false  (LOW) на входах означает, что внешний сигнал притянул вывод к земле.


  ### 6. Пример реализации в коде [ВЫПОЛНЕНО]

  Добавьте следующий код в ваши файлы проекта:

    #include <Servo.h>

    Servo incubatorServo; // Создаем объект управления сервоприводом

    void setup() {
      // Инициализируем сервопривод на GPIO15.
      // Стандартные параметры ширины импульса для большинства сервоприводов: 544 мкс (0°) - 2400 мкс (180°).
      incubatorServo.attach(15);

    }

    void loop() {
      // Пример поворота на 0 градусов
      incubatorServo.write(0);
      delay(2000);

      // Пример поворота на 180 градусов
      incubatorServo.write(180);
      delay(2000);
    }

  ### 7. Полезные функции библиотеки:

  •  incubatorServo.attach(pin, min, max)  — привязка к пину с явным указанием минимальной и максимальной длины
  импульса (в микросекундах) для точной калибровки углов.
  •  incubatorServo.write(angle)  — поворот на угол от  0  до  180  градусов.
  •  incubatorServo.writeMicroseconds(us)  — прямое управление длительностью импульса (обычно от  1000  до  2000
  мкс). Позволяет позиционировать сервопривод точнее, чем в градусах.
  •  incubatorServo.detach()  — отключает генерацию сигнала управления. Полезно для снятия напряжения с мотора
  сервопривода, чтобы он не гудел и не расходовал энергию, когда не двигается.

  Анализ использования памяти вашей прошивки для ESP8266 (ESP-12E):

  ### 1. Оперативная память (RAM)

  • Занято статически:  36 896 байт  (45%) — это память под глобальные переменные и внутренние буферы библиотек.
  • Доступно для кучи (Heap): около  45 024 байт  (55%).
  • Вывод: Запас в 44 КБ свободной RAM — отличный показатель для ESP8266. При запуске HTTPS-соединений (Telegram
  использует SSL/TLS) библиотека  BearSSL  динамически забирает от 5 до 12 КБ RAM под буферы шифрования.
  Дополнительно память требуется для  WiFiManager  и Web-сервера. Имеющегося запаса RAM более чем достаточно для
  стабильной работы без сбоев из-за нехватки памяти (Out of Memory).

  ### 2. Флеш-память (Flash)

  • Занято:  579 495 байт  (55.5%) от общего лимита на размер бинарного файла в  1 044 464 байт  (~1 МБ).
  • Свободно:  464 969 байт  (~454 КБ).
  • Вывод:
      • Программа занимает чуть больше половины выделенной зоны под код. Места для расширения функционала
      предостаточно.
      • Физически на ESP-12E распаяно 4 МБ Flash. Текущие лимиты обусловлены схемой разметки памяти в  platformio.ini
      (остальная память отдана под файловую систему LittleFS и резерв для OTA-обновлений).
      • Для поддержки стандартного двухзонного OTA (беспроводного обновления) размер текущей прошивки ( 579 КБ )
      немного превышает свободный остаток ( 464 КБ ) в текущей схеме разметки. Если планируется использовать OTA,
      может потребоваться настроить разметку в  platformio.ini  (например, выбрав схему с меньшим размером LittleFS и
      большим размером зоны OTA).

> что такое OTA-обновление

▸ Thought for 2s, 351 tokens
  Defining OTA Updates
  OTA-обновление (Over-the-Air, «по воздуху») — это технология беспроводной загрузки новой прошивки в микроконтроллер
  через Wi-Fi, без необходимости физического подключения устройства к компьютеру через USB-кабель.

  ### Как это работает на ESP8266:

  1. Вы компилируете проект на компьютере и получаете готовый бинарный файл прошивки ( firmware.bin ).
  2. ESP8266, находясь в локальной сети Wi-Fi, запускает внутренний OTA-сервер (например, через библиотеку
  ArduinoOTA  или веб-страницу на встроенном веб-сервере).
  3. Вы отправляете файл  firmware.bin  на IP-адрес ESP8266 (через Arduino IDE/PlatformIO или загрузив его через веб-
  интерфейс в браузере).
  4. ESP8266 принимает файл, параллельно записывая его во временный свободный сектор флеш-памяти.
  5. После успешного получения и проверки целостности файла (контрольной суммы), ESP8266 меняет указатель загрузчика
  на новую прошивку и автоматически перезагружается.

  ### Важное ограничение по памяти:

  Поскольку во время загрузки старая прошивка должна работать (поддерживать соединение Wi-Fi и принимать данные),
  ESP8266 не может сразу перезаписывать свой активный код.

  Для этого флеш-память делится на две равные части:

  • Активная зона: где сейчас выполняется программа.
  • Зона OTA: куда принимается новый файл.

  Поэтому размер файла прошивки не должен превышать размер свободной зоны OTA. На ESP-12E с 4 МБ памяти это легко
  настроить, выделив, например, 1 МБ под прошивку, 1 МБ под OTA и остальное под файлы LittleFS.

  Необходимые изменения в коде:

  1. В начале файла  src/main.cpp  (или там, где инициализируется веб-сервер) подключите библиотеку:
    #include <ESP8266HTTPUpdateServer.h>

  ### Сценарий 2. Автоматическое обновление по сети (Auto-Update)

  Используется, если плата должна сама скачивать прошивку с вашего сервера (например, по команде из Telegram-бота или
  по таймеру).

  Необходимые изменения в коде:

  1. В файле  src/telegram.cpp  (или другом месте управления логикой) подключите библиотеки:
    #include <ESP8266httpUpdate.h>
    #include <WiFiClient.h>

  2. Напишите функцию для запуска обновления:
    void performFirmwareUpdate() {
      WiFiClient client;

      MYDEBUG_PRINTLN("Запуск скачивания обновления...");

      // Отключаем WDT на время загрузки, чтобы плата не перезагрузилась в процессе приема файла
      ESP.wdtDisable();

      // Указываем URL вашего сервера с файлом прошивки
      t_httpUpdate_return ret = ESP8266httpUpdate.update(client, "http://your-server-domain.com/firmware.bin");

      // Включаем WDT обратно в случае сбоя
      ESP.wdtEnable(5000);

      switch (ret) {
        case HTTP_UPDATE_FAILED:
          MYDEBUG_PRINTLN("Ошибка обновления: " + ESP8266httpUpdate.getLastErrorString());
          bot.sendMessage(chatID, "Ошибка обновления!", "");
          break;
        case HTTP_UPDATE_NO_UPDATES:
          MYDEBUG_PRINTLN("Обновлений на сервере нет.");
          break;
        case HTTP_UPDATE_OK:
          MYDEBUG_PRINTLN("Обновление завершено! Перезагрузка...");
          break;
      }
    }

  3. Вы можете вызывать эту функцию при получении команды  /update  в Telegram:
    if (bot.messages[i].text == "/update") {
      bot.sendMessage(chatID, "Начинаю обновление прошивки...", "");
      performFirmwareUpdate();
    }
